# ADR

ADR (Architecture Decision Record) — короткая запись об архитектурном решении: контекст, выбор, альтернативы, последствия и условие пересмотра. ADR пишется, когда решение влияет на структуру системы, данные, инфраструктуру, безопасность, надёжность, стоимость или скорость разработки.

## Статусы

- `Предложено` — решение обсуждается.
- `Принято` — решение действует.
- `Отменено` — решение больше не применяется.
- `Заменено` — решение заменено другим ADR.
