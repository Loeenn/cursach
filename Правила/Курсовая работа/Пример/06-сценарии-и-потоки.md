# 06. Сценарии и потоки

## Создание задания и получение результата

```mermaid
sequenceDiagram
    actor User as Пользователь
    participant TG as Telegram Platform
    participant BOT as telegram-bot-adapter
    participant API as control-plane-api
    participant DB as MongoDB
    participant Q as RabbitMQ
    participant PP as preprocessing-worker
    participant SYN as synthesis-worker
    participant ASM as assembly-worker
    participant S3 as Object storage

    User->>TG: Отправить текст, URL или файл
    TG->>BOT: Update от Telegram Bot API
    opt Входной файл
        BOT->>TG: Скачать файл по file_id
    end
    BOT->>API: Создать задание от telegram_user_id
    API->>S3: Сохранить исходный файл или payload
    API->>DB: Создать Job со статусом created
    API->>Q: job.preprocess.requested
    API-->>BOT: job_id
    BOT->>TG: Сообщить пользователю, что задание создано
    TG-->>User: job_id и начальный статус

    PP->>Q: Получить job.preprocess.requested
    PP->>S3: Прочитать исходник
    PP->>PP: Извлечь, нормализовать, разбить на фрагменты
    PP->>S3: Записать preprocessing manifest
    PP->>DB: Сохранить quota estimate и awaiting_confirmation
    PP->>Q: job.preprocess.completed

    BOT->>API: Получить оценку
    API->>DB: Прочитать quota estimate и подписку
    API-->>BOT: Оценка квоты и параметры задания
    BOT->>TG: Отправить оценку квоты и кнопку подтверждения
    TG-->>User: Оценка квоты и параметры задания
    User->>TG: Подтвердить синтез
    TG->>BOT: Callback query подтверждения
    BOT->>API: Подтвердить синтез
    API->>DB: Проверить активную подписку и остаток недельной квоты
    alt Подписка активна и квоты достаточно
        API->>DB: Зарезервировать estimated_audio_seconds
        API->>DB: queued_for_synthesis
        API->>Q: job.synthesis.requested batches в очередь тарифа
    else Нет подписки или квоты
        API-->>BOT: Ошибка подтверждения
        BOT->>TG: Попросить оформить подписку или дождаться обновления квоты
    end

    SYN->>Q: Выбрать priority/standard queue по весу 70/30
    SYN->>S3: Прочитать manifest
    SYN->>SYN: Синтезировать WAV-фрагменты
    SYN->>S3: Записать batch archive
    SYN->>DB: Обновить прогресс

    ASM->>Q: Получить job.assembly.requested
    ASM->>S3: Прочитать batch archives
    ASM->>ASM: Собрать m4b или opus
    ASM->>S3: Записать итоговый артефакт
    ASM->>DB: Зафиксировать output artifact и delivery_mode
    ASM->>Q: job.delivery.requested

    BOT->>Q: Получить job.delivery.requested
    BOT->>API: Получить параметры выдачи
    API->>DB: Проверить владельца, статус и size_bytes
    alt size_bytes <= 50 МБ
        API-->>BOT: object_key для Telegram voice
        BOT->>S3: Прочитать opus artifact
        BOT->>TG: sendVoice
        TG-->>User: Голосовое сообщение
        BOT->>API: Зафиксировать telegram_file_id
    else size_bytes > 50 МБ
        API->>S3: Создать presigned URL на 30 дней
        API-->>BOT: Ссылка на presigned URL
        BOT->>TG: Отправить ссылку
        TG-->>User: Ссылка на скачивание
    end
```

## Подписка и отписка

```mermaid
sequenceDiagram
    actor User as Пользователь
    participant TG as Telegram Platform
    participant BOT as telegram-bot-adapter
    participant API as control-plane-api
    participant PAY as Платежный провайдер
    participant DB as MongoDB

    User->>TG: /subscribe basic или priority
    TG->>BOT: Update от Telegram Bot API
    BOT->>API: Создать подписку выбранного тарифа
    API->>PAY: Создать checkout или subscription
    PAY-->>API: payment_session_id и ссылка оплаты
    API->>DB: Сохранить pending subscription
    API-->>BOT: Ссылка оплаты
    BOT->>TG: Отправить кнопку оплаты
    TG-->>User: Checkout link
    PAY->>API: Webhook recurring payment succeeded
    API->>DB: subscription active, tariff, period_end
    BOT->>API: Получить обновленный статус подписки
    BOT->>TG: Сообщить, что подписка активна

    User->>TG: /unsubscribe
    TG->>BOT: Update от Telegram Bot API
    BOT->>API: Отменить подписку
    API->>PAY: Cancel subscription
    PAY-->>API: cancellation accepted
    API->>DB: cancel_at_period_end или cancelled
    API-->>BOT: Статус отмены
    BOT->>TG: Подтвердить отмену подписки
```

## Жизненный цикл задания

```mermaid
stateDiagram-v2
    [*] --> created
    created --> preprocessing: задача опубликована
    preprocessing --> awaiting_confirmation: manifest готов
    preprocessing --> failed: ошибка предобработки
    awaiting_confirmation --> queued_for_synthesis: пользователь подтвердил
    awaiting_confirmation --> quota_rejected: нет подписки или квоты
    quota_rejected --> awaiting_confirmation: подписка активирована или квота обновлена
    awaiting_confirmation --> cancelled: пользователь отменил
    queued_for_synthesis --> synthesizing: первый batch взят в работу
    synthesizing --> assembling: все batch готовы
    synthesizing --> failed: исчерпаны повторы синтеза
    synthesizing --> cancelled: отмена до сборки
    assembling --> ready_for_delivery: итоговый артефакт готов
    assembling --> failed: ошибка сборки
    ready_for_delivery --> delivering: delivery-задача взята
    delivering --> delivered: voice или ссылка отправлены
    delivering --> ready_for_delivery: временная ошибка Telegram
    delivering --> failed: исчерпаны повторы выдачи
    delivered --> [*]
    failed --> [*]
    cancelled --> [*]
```

## Правила повторов

- Повторное сообщение из RabbitMQ должно сверяться с текущим состоянием Job в MongoDB.
- Если стадия уже завершена, worker не должен выполнять ее повторно без явной причины.
- Batch synthesis можно повторять на уровне отдельного batch, если итоговый batch archive не был успешно зафиксирован.
- Assembly запускается только после того, как все ожидаемые batch results зарегистрированы.
- Резервирование недельной квоты выполняется атомарно при подтверждении задания; при окончательном отказе синтеза резерв освобождается или корректируется.
- Повторный webhook платежного провайдера должен обрабатываться идемпотентно по external event id.
