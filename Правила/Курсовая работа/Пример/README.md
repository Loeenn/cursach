# Пример

Этот каталог показывает, как можно заполнять шаблон курсовой работы на примере системы синтеза речи. Пример описывает целевую архитектуру MVP, а не состояние готового кода.

Система позволяет пользователю оформить подписку, отправить книгу, статью, ссылку или текстовое сообщение в Telegram-бота и получить аудио, сгенерированное встроенным ONNX Runtime. Основной акцент MVP - backend pipeline: прием задания через Telegram, проверка подписки и недельной квоты, предобработка текста, оценка объема, синтез, сборка итогового аудио и выдача результата.

## Статус примера

Документация находится в первой итерации. Она задает каркас, основные решения и базовые схемы. На следующих итерациях разделы можно углублять: добавлять больше сценариев, уточнять модель данных, расширять ADR, описывать отказоустойчивость, безопасность и тестовую стратегию.

## Основные решения первой итерации

- Система строится как асинхронный pipeline.
- Основной пользовательский интерфейс MVP - Telegram-бот.
- Монетизация строится на месячной рекурентной подписке с двумя тарифами: базовым и приоритетным.
- Базовый тариф дает 30 часов аудио в неделю, приоритетный тариф - 60 часов аудио в неделю.
- Приоритетный тариф ускоряет обработку за счет отдельной очереди synthesis: worker выбирает приоритетную очередь с вероятностью 70%, обычную - с вероятностью 30%; если приоритетных задач нет, все worker обрабатывают обычную очередь.
- Control plane отделен от worker-процессов.
- RabbitMQ доставляет задачи, MongoDB хранит состояние, S3-совместимое хранилище хранит файлы и артефакты.
- Предобработка текста завершается до синтеза, потому что оценка потребления недельной квоты строится по нормализованному тексту.
- Синтез выполняется пакетами, чтобы снизить накладные расходы и упростить повторы.
- Итог до 50 МБ отправляется пользователю как Telegram voice message, более крупный итог выдается ссылкой на presigned S3 URL сроком на 30 дней.
