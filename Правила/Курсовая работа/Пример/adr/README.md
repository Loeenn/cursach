# ADR примера

В этом каталоге фиксируются архитектурные решения TTS-системы. ADR не заменяют основные разделы документации, а объясняют, почему выбрано конкретное решение и какие компромиссы приняты.

- [ADR-0002: Разделить control plane и worker-процессы](0002-разделить-control-plane-и-workers.md)
- [ADR-0003: Использовать RabbitMQ, MongoDB и S3](0003-использовать-rabbitmq-mongodb-и-s3.md)
- [ADR-0004: Использовать подписки и взвешенные synthesis-очереди](0004-использовать-подписки-и-взвешенные-synthesis-очереди.md)

## Правило для следующих итераций

Если новое решение меняет границы сервисов, модель данных, инфраструктуру, надежность, безопасность, стоимость или способ эксплуатации, оно должно получить отдельный ADR.
